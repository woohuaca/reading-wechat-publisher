# Reading Input Modes

Use this file before drafting so the workflow makes the right promise.

## Mode 1: 导读笔记

Use this when the user provides only:

- book title
- author
- public reviews
- publisher blurbs
- catalog pages
- sparse excerpts

### What this mode may do

- explain the book's likely core problem
- reconstruct the public framing
- compare it with adjacent PM, management, or strategy ideas
- create a useful guide for whether the book is worth reading

### What this mode must not do

- fake chapter-by-chapter certainty
- invent direct quotes
- claim page-level evidence that was never seen
- present public inference as if it came from full-book reading

## Mode 2: 精读笔记

Use this when the user provides substantial primary material:

- full `pdf`, `epub`, or `txt`
- dense highlights
- chapter screenshots
- detailed excerpts
- spoken reflections tied to specific parts of the book

### What this mode should do

- preserve chapter sequence where helpful
- capture exact terms from the book
- note page or chapter anchors when available
- distinguish author claims from your interpretation

## Mixed Inputs

Sometimes the user provides partial primary material plus public context.

In that case:

- keep the overall mode as `导读笔记` unless the primary material is substantial
- add a section such as `我目前能合理确认的范围`
- keep unsupported claims out of the “facts” lane

## Output Promise

The `source_mode` name is not a cosmetic label. It is an honesty contract for:

- certainty level
- evidence density
- quote safety
- how hard the article may lean on interpretation

It does not decide article tone by itself.  
Tone and framing should be chosen separately via `writing_mode`.
