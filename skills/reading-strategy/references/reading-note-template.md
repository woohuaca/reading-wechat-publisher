# Reading Note Template

Use this structure when creating the first markdown artifact.

```md
---
title: "书名"
book_title: "书名"
book_author: "作者"
core_phrase: "一句核心词"
source_mode: "精读笔记"
writing_mode: "结构导读"
source_material: "pdf / highlights / excerpts / title-only"
---

# 书名

## 一句话总述

用 2-4 句说明这本书的主张、视角和读完后的变化。

## 这本书在解决什么问题

- 它想回答什么关键问题
- 这个问题为什么值得回答
- 作者站在什么立场

## 核心框架

用 3-6 条讲清作者的方法、模型、因果链或判断标准。

## 关键洞见

写 3-5 条最值得记住的观点。

## 反直觉点

列出最容易被忽略、误读、或与常识冲突的地方。

## 章节拆解

当你确实有章节素材时再写。没有就删除此节，不要虚构。

## 金句与证据

- 引文 1
- 引文 2
- 数据 / 案例 / 实验

## 行动建议

把书里的洞见改写成读者可执行的动作。

## 局限与争议

说明作者没有覆盖、可能夸大、或证据不足的地方。

## 公众号改写线索

- 适合做标题的冲突点
- 适合做知识卡的框架
- 适合做开头故事的情节
```

If only title and author are available, rename `source_mode` to `导读笔记` and replace unsupported certainty with informed, clearly labeled interpretation.

If `writing_mode` is `诊断模式`, prefer adding:

- `时代症候`
- `它击穿了什么幻觉`
- `隐藏机制`
- `认知断层`
- `现实拷问`

If `writing_mode` is `战略模式`, prefer adding:

- `战略含义`
- `决策映射`
- `适用边界`
- `下一步动作`
