# Reading Writing Modes

Use this file after choosing `source_mode`.  
`source_mode` is about honesty and evidence.  
`writing_mode` is about how the material is organized for the reader.

## Mode A: 结构导读

Use this when the user wants:

- a clean introduction
- a framework map
- a balanced reading note
- a first-pass source draft for later editing

### What this mode emphasizes

- the book's core question
- the main framework
- the most useful insights
- clear, even pacing

## Mode B: 诊断模式

Use this when the user wants the book to explain a larger时代问题.

### Best-fit situations

- strategy, geopolitics, management, social diagnosis, technology shifts
- books that break a mainstream illusion
- content intended for public commentary, WeChat, or leadership reading

### What this mode should do

- start from a时代症候 or live tension
- name the illusion the book breaks
- surface 3-5 hidden mechanisms
- create a clear `大众认知 vs 书中揭示` contrast
- connect the book to larger intellectual or industry narratives
- end with现实拷问 or pressure-test style questions

### What this mode must avoid

- pretending the book said things the book did not say
- fake quotes or dramatized citations
- replacing evidence with tone

## Mode C: 战略模式

Use this when the user wants the book mapped to a concrete decision arena.

### Best-fit situations

- product strategy
- enterprise AI
- investment cases
- policy design
- operating model redesign

### What this mode should do

- translate the book into decision criteria
- map the insight to the user's business or operating context
- separate what is diagnosis from what is recommendation
- make tradeoffs explicit

## Practical Rule

When in doubt:

- keep `source_mode` conservative
- choose `writing_mode` aggressively only after the evidence lane is clean
